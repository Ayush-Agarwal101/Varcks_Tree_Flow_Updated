# Project Specification

## User Requirement
build a backend for online bakery shop

## Selected Technology Stack

### 1. Web Development

### 2. Backend
**Why chosen:** Needed to manage bakery shop data and operations.
**Purpose:** Handles server-side logic, databases, API endpoints for the bakery shop.

### 3. REST
**Why chosen:** Simple and widely used for backend APIs.
**Purpose:** Enables easy creation of API endpoints for bakery shop functionalities.

### 4. Python
**Why chosen:** Elegant and easy to use for API development.
**Purpose:** Enables rapid development of RESTful APIs with frameworks like Flask or Django.

### 5. Django
**Why chosen:** Comprehensive framework with built-in features for authentication, content management system, and database migrations.
**Purpose:** Enables rapid development and maintenance of the backend by leveraging a robust set of tools and conventions.

### 6. NGINX
**Why chosen:** reverse proxy and load balancer
**Purpose:** to handle static files, SSL termination, and provide a performance boost

## Technology Stack Summary
Web Development → Backend → REST → Python → Django → NGINX
